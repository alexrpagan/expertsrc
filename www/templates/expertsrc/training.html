First, what is data curation?

Data curation is the process of maintaining a collection of data sources over time.


Common data curation tasks include:


  • Importing relevant data sources into a collection
  • Transforming and cleaning imported data
  • Semantically integrating imported data
  • Removing duplicate entries from integrated sources

We would like you to help us with semantically integrating imported data.

This step is called schema integration.


During schema integration, the data curator matches data attributes from imported sources (which we call local attributes) with global data attributes within the collection.


The curator make a match if the data in the local and global attributes have the same semantic type.

Okay, so what is Data Tamer?

Data Tamer is a data curation system.


Data Tamer uses machine learning algorithms to complete data curation tasks automatically and at scale.

Data Tamer is human-aided.

When we suspect that our algorithms might be wrong about a data curation task, Data Tamer calls in a human domain expert (or a group thereof) to help complete the task.

You can help us improve Novartis's Pharon assay data (and our system!) by completing some simple schema integration tasks over the course of the next 4 weeks.

How it works:

  • Each week, we will run between one and four assay data sources from your research department through Data Tamer's algorithms.
  • From these sources, we will generate a set of schema mapping tasks, which will be assigned to domain experts in your group, including yourself. You will be notified by email when you have been assigned to new questions.

The tasks

  • For each task, you will be asked whether an attribute from a local source and an attribute from the global schema contain the same sort of data.
  • The global attribute given in the task is Data Tamer's best guess at a match.
  • To answer, you either reply Yes, No, or specify a global attribute which matches better.

Please note!

  • Our global attributes are a combination of result-typing attributes from the MAGMA system and metadata attributes from ARES. It is a currently incomplete collection.
  • Hence, your answers to schema mapping tasks are regarded as best-effort. We recognize that schema mapping questions do not always admit a clear best answer.

Also...

  • You will never be penalized for answering a question incorrectly.
  • We do not intend for you to spend more than 60 minutes per week on these tasks.
